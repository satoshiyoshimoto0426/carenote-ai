# P2 調査メモ: カイポケ連携（時短エンジン）

> 作成: 2026-06-11 / 吉本決定「P2の対象介護ソフトは一旦カイポケ」を受けた事前調査（SPEC §7.2 規約確認・R2対応）。
> 本書は調査時点のWeb公開情報に基づく。**実装着手前に最新の規約・画面で再確認すること。**

## 1. 確定事項

- P2（ブラウザ拡張による自動入力）の対象介護ソフトは**カイポケ**（株式会社エス・エム・エス）とする（一旦。コンサル先の実態次第で見直し可）。

## 2. 調査結果（2026-06-11 時点）

| 項目 | 結果 | 出典 |
|---|---|---|
| 提供形態 | クラウド型SaaS（介護・障害福祉・訪問看護向け経営支援） | 公式 ads.kaipoke.biz |
| 推奨ブラウザ | Chrome / Edge 最新版（= **Chrome拡張(MV3)と整合**） | Web検索結果（公式・解説サイト） |
| 規約上の自動化禁止 | カイポケ利用規約（policy.bm-sms.co.jp の カイポケクラウド規約）に、スクレイピング・自動化プログラム・ブラウザ拡張・リバースエンジニアリングの**明示的な禁止条項は確認できなかった** | policy.bm-sms.co.jp |
| 規約の注意点 | ただし規約は複数文書（クラウド/モバイル/タブレット等）に分かれ、「当社が不適切と判断した行為」型の一般条項が存在。**断定はせず、本格運用前にSMS社への確認を推奨** | 同上 |
| 公開API | 公開された開発者向けAPIは**確認できず** | Web検索 |
| 公式のデータ連携 | ①**ケアプランデータ連携システム**（国保中央会）対応＝標準仕様CSVの出力・取込が可能（ベンダ試験V4対応） ②カイポケ会員事業所同士の「ケア連携」（提供票等のオンライン授受） | ads.kaipoke.biz / kokuho.or.jp |

## 3. P2 実装方針（段階制・リスク最小化）

```
Step 1（DOM非依存・規約リスクほぼゼロ）――――――――――――― すぐ作れる
  Chrome拡張のサイドパネルに CareNote AI の下書きをフィールド単位で表示し、
  ワンクリックコピー → 利用者がカイポケの欄に貼り付け（タブ切替なしの転記支援）。
  ・画面の読み取りも自動操作もしない＝どのソフトでも動く汎用転記支援
  ・「コピペ作業の往復」を消すだけでも時短効果あり

Step 2（カイポケ・アダプタ＝半自動入力）――――― 実画面の確認後に着手
  カイポケの帳票画面のDOM構造を実環境で確認し、adapters/kaipoke を実装。
  下書きを該当欄へ自動入力 → 人が確認して保存（SPEC F7: 全自動確定はしない）。
  ・前提: 吉本さん（またはコンサル先）のカイポケ環境で帳票画面を確認できること
  ・DOM構造は推測で書かない（ハルシネーション禁止 §2.6-A）

代替路線（Step 2が規約・技術面で詰まった場合）
  ケアプランデータ連携システムの標準仕様CSVを CareNote AI から出力する形を検証。
  ※同システムの本来用途は事業所間の授受であり、外部ツール起点の取込可否は要検証。
```

## 4. リスク・未確認事項

| # | 内容 | 対応 |
|---|---|---|
| K1 | 規約の全文書（共通規約・別紙含む）は未精査。「不適切と判断した行為」条項の解釈 | Step 2 着手前にSMS社へ照会するか吉本さんが判断（Step 1 は読取・操作をしないため低リスク） |
| K2 | カイポケ帳票画面のDOM構造が未確認 | 吉本さんのログイン環境で確認（スクリーンショット＋開発者ツールのHTML）。デモ・無料体験の利用も選択肢 |
| K3 | 個人情報の取り扱い | 拡張はローカル処理のみ（下書きの表示・コピー）。**カイポケ画面の内容を外部送信しない**設計とする（SPEC §12 Never 準拠） |
| K4 | ケアプランデータ連携CSVの仕様入手・外部ツールからの利用可否 | 代替路線が必要になった時点で国保中央会の仕様書を確認 |

## 5. 吉本さんへの依頼（Step 2 の前提）

1. カイポケに**ログインできる環境**があるか（自身の事業所 / コンサル先 / デモ）
2. あれば: ケアプラン第1・2表あたりの**入力画面のスクリーンショット**（個人情報はマスキング）
3. SMS社への事前照会を「する/しない」の判断（Step 1 は照会不要で進められる見込み）
