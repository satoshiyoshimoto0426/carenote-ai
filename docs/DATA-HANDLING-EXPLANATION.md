## まず1枚サマリ（詳細は各節）

| 観点 | 答え |
|---|---|
| 何が外部に出るか | 記号化した記録テキスト（登録利用者の氏名は自動で「A様」等に置換）と、添付PDF（**PDFは原本のまま**＝実名を含む） |

## 1. この資料の目的

CareNote AI（介護記録の作成補助サービス）の導入にあたり、貴法人の利用者様の個人情報——とくに
**要配慮個人情報**（心身の状態・介護の記録など）——を**どのような仕組みで、どこまで、どう守って**
扱うのかをご説明するものです。個人情報保護法の**委託先の監督**（法第25条）のための資料としても
お使いいただけます。

## 2. 法的な位置づけ（だれが・どの立場か）

| 立場 | 主体 | 役割 |
|---|---|---|
| 委託元（個人情報取扱事業者） | 貴法人 | 利用目的の範囲内で記録作成業務に本サービスを利用 |
| 委託先 | 吉本（CareNote AI） | 記録の下書き作成の**業務委託**として個人情報を取り扱う |
| 再委託先 | Anthropic, PBC（米国） | AI（Claude API）による文章生成処理 |
| 〃 | Supabase Inc.（米国） | 保存書類・利用者情報（実名は暗号化）のデータベース保管 |
| 〃 | Vercel Inc.（米国） | アプリの配信・添付PDFの一時保管（処理後削除） |
| 〃 | Clerk Inc.（米国） | 職員アカウントの認証（職員の氏名・メールアドレス） |

- 業務委託に伴う個人データの提供は「第三者提供」に該当しません（法第27条5項1号。
  **利用目的の達成に必要な範囲内に限る**）。国内の委託であれば利用者本人の同意は不要とされています。
- **ただし本サービスは国外（米国）での処理・保管を含む**ため、外国にある第三者への提供の規律
  （法第28条）が別途適用されます。対応は「§6 国外処理について」をご覧ください。

## 3. データの流れ（何が・どこへ行くか）

```
① 職員がメモ・資料を入力
     ・登録済み利用者の氏名は自動で記号化（例:「山田花子」→「A様」）してからAIへ
     ・実名そのものは暗号化して別領域に保管（AIへは渡さない）
        ↓
② AI（Claude API・米国 Anthropic社）が帳票の下書きを生成
        ↓
③ 職員が画面で内容を確認 → 保存した書類は「承認」ボタン（誰が・いつ承認したか記録に残る）
        ↓
④ 職員が介護ソフト（カイポケ）へ転記して確定（確定は常に人間）
```

**大切な例外を正直に**：
- **添付PDF（診療情報提供書・認定調査票など）は原本のまま**AIへ送信されます（実名・病名を含む）。
  PDFは読解処理の後すみやかに削除し、AI側でも30日以内に自動削除・学習利用は契約で禁止されています。
- 氏名の自動記号化が働くのは**本サービスに登録済みの利用者の氏名**です。登録していない方の実名や
  あだ名などは置換できません。そのため**「メモに実名を書かない」を第一の運用ルール**とし、
  自動記号化はその安全網と位置づけています（§7）。

## 4. 当方（CareNote AI）が実装している安全管理措置

| 対策 | 内容 |
|---|---|
| 仮名化 | 登録利用者の氏名を記号（A様等）へ自動置換してからAIへ送信。実名はAES-256-GCM（政府機関等でも用いられる強度の暗号化方式）で暗号化し分離保管 |
| 人間承認 | AI出力は常に「下書き」。**保存した書類**は、職員の承認操作（承認者・日時を記録）を経るまでコピー機能が使えない。生成直後の画面にも「下書き・要事実照合」を常時表示 |
| 添付資料の非保持 | アップロードされたPDFは一時保管ののち、処理後**すみやかに削除**（恒久保存しない） |
| アクセス制御 | ログイン（または発行制トークン）必須。保存書類は作成した職員本人のアカウントに紐づく。※当方（提供者）は保守のため技術的にアクセス可能ですが、障害対応等の必要時に限ります |
| 保持期間 | 保存書類に期限（作成から5年）を記録。**期限経過後の自動削除機能は開発中**で、それまでは求めに応じ手動で削除対応。介護記録の保存義務の起算（完結日基準）や貴法人規程との整合は導入時に協議のうえ設定 |
| 通信の保護 | すべての通信はTLS（インターネット通信を暗号化する標準技術・ブラウザの鍵マークの仕組み）で保護 |

## 5. AI事業者（Anthropic）側の保護（出典つき・公開文書）

AI処理の再委託先である Anthropic 社との間には、**データ処理補遺（DPA）が自動的に適用**されています
（利用規約に組み込み済み・公開文書のため貴法人でも直接ご確認いただけます）。

| 項目 | 内容 | 出典 |
|---|---|---|
| 学習への利用 | 入力・出力を**AIの学習に使うことは契約で禁止** | anthropic.com/legal/commercial-terms（§B） |
| 保存期間 | APIの入力・出力は**30日以内に自動削除**（規約違反としてフラグされた場合等の例外あり。詳細は別紙照会記録） | privacy.claude.com ヘルプ記事 7996866 |
| 契約上の役割 | 当方＝管理者（controller）／Anthropic＝処理者（processor）※ | anthropic.com/legal/data-processing-addendum |
| 終了時 | 契約終了後30日以内にデータを返却または削除 | 同上 |
| 事故時 | セキュリティ侵害は**48時間以内に**当方へ通知 | 同上 |
| 技術的安全管理 | DPA Schedule 2（技術的・組織的安全管理措置）に規定 | 同上 |

※「管理者／処理者」は Anthropic との契約（DPA・欧州GDPR系の用語）上の呼称で、「Anthropicは当方の指示の
範囲でのみ処理する」ことを意味します。貴法人・当方・各社の日本法上の関係は §2 の整理のとおりです。

## 6. 国外処理について（法第28条への対応）

- AI処理・データ保管等は米国の事業者（§2の表）で行われます（※各社の詳細な処理拠点・リージョンは
  確認中の項目があり、確定次第お知らせします）。
- 対応方針（いずれかを貴法人と協議のうえ確定します）:
  1. **本人同意方式（推奨）**: 重要事項説明書・個人情報利用同意書に外国での処理を明記し同意を得る。
     ※法第28条では、同意取得の際に「**どの国か・その国の個人情報保護制度・提供先の保護措置**」の
     情報提供が必要とされています。この要件を満たす同意書ひな型は当方で用意し、**専門家の確認を経て**提供します。
     ※認知症等で同意能力に課題がある利用者様については、ご家族・成年後見人への説明を含む運用を協議します。
  2. **相当措置方式**: 各社の保護措置が法の求める基準に適合する体制に該当するかを確認のうえ同意に代える。
     **この方式では、相当措置が継続されているかの定期確認と、本人の求めに応じた情報提供が継続的に
     必要になります**（採否は専門家の確認を推奨）。
- いずれの場合も、貴法人のプライバシーポリシー／利用目的の掲示への「外部委託（国外処理を含む）」の
  記載追加をお願いします（文案は当方で用意します）。

## 7. 貴法人にお願いすること（運用ルール）

1. **実在の利用者情報の投入は、本説明の確認と合意書の締結後から**（それまではテストデータのみ）
2. **メモ・入力欄に利用者の実名を書かない**（利用者は先に登録し、記号で呼ぶ。自動記号化は安全網）
3. AI出力は**必ず内容を確認し、承認操作をしてから**使用する運用の徹底
4. 職員アカウントの適切な管理（パスワード・トークンを共有しない。退職時は当方へ連絡→即時停止）
5. §6 の対応方式（同意方式／相当措置方式）の選択
